# 第14章：系统设计与玩家分类（技巧57-62）

## 技巧57：系统的状态模型与少即是多
- 系统复杂度需精简，状态数量要控制
- 过多状态→玩家认知负担过重
- **少即是多**：用最少的系统状态表达最丰富的玩法

## 技巧58：五层系统的模型与庄子说剑
系统分五层（由内到外）：
1. **核心层**：不可删除，删除即失去游戏本质
2. **外延层**：增强核心体验，高度关联
3. **辅助层**：辅助核心体验，可独立存在
4. **装饰层**：锦上添花，有则更好
5. **冗余层**：可直接删除，不影响体验

**庄子说剑**：好剑只有一个锋利的刃，而非全身都是刃。

## 技巧59：系统深入设计的流程与推陈出新
1. 明确系统的核心目的
2. 设计最小可行系统
3. 测试核心循环是否成立
4. 逐层向外扩展
5. 每扩展一层，验证是否增强了核心体验
6. 不增强则删除

## 技巧60：玩家分类模型与新旧竞争
- 新玩家和老玩家的需求天然矛盾
- 新玩家需要简单、引导、成就感
- 老玩家需要深度、新内容、社交地位
- 设计需同时兼顾，用分层内容解决
- 新玩家的涌入会改变游戏生态

## 技巧61：换皮设计流程与一以贯之
- 换皮的核心：保留已验证成功的底层机制
- 更换：题材、美术、故事、表现
- 保留：核心循环、数值框架、关键体验
- **一以贯之**：核心体验不变，外在表现更新

## 技巧62：换皮执行技巧与返璞归真
- 换皮时先回到**机制本质**
- 去掉所有表现层装饰，看纯机制是否有趣
- 在纯机制基础上重新包装
- 避免换皮过程中误改核心机制
